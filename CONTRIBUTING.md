# Contributing to Finance MCP

Finance MCP is a maintained project that powers financial data retrieval, document analysis, and credit assessment workflows.

## When to open a pull request

An issue does not automatically imply that a pull request is required.

### Bug fixes

Pull requests are welcome for well-scoped bug fixes where:

* The root cause is understood
* The fix is straightforward
* The change has limited impact on surrounding systems

Examples include:

* Incorrect tool routing
* Parsing failures
* Missing validation checks
* Configuration defects


### Enhancements

For enhancements, workflow changes, new MCP tools, extraction strategies, or architectural modifications, discuss the proposal in an issue before implementation.

Enhancements should explain:

* The business problem being solved
* The expected user impact
* How the change fits within existing Finance MCP patterns


## Pull Request Guidelines

If you open a PR:

* Reference the related issue.
* Keep changes focused on a single logical improvement.
* Add or update tests where applicable.
* Address root causes instead of introducing workaround-based fixes.

## Final Notes

Finance MCP supports business-critical financial workflows. Successful contributions improve reliability, maintainability, and consistency across tools and services. When in doubt, start with a well-written issue describing the problem and its business impact before proposing code changes.
